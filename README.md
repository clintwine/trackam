# Web App Template

`templates/web_app` is a Phase 0 workspace/account/admin scaffold.

Current runtime coverage:
- auth
- users
- roles
- notifications
- events
- devices
- sessions
- settings
- user and admin dashboards

Current stack:
- backend: Express 5 + Postgres + raw SQL repositories
- frontend: React 19 + Vite + TypeScript + React Router

## Local demo accounts

Running `backend/scripts/seedLocal.js` through `npm run db:seed` creates two local demo users:

- Admin: `admin@example.com` / `password123`
- User: `user@example.com` / `password123`

The seed is idempotent, so rerunning `npm run db:seed` updates those records instead of duplicating them.

Scaffolded projects render `backend/.env` and `frontend/.env.local` with their local runtime values, so local `npm run dev` and `npm run start` should work without manual env patching.

## Remote bootstrap admin

- `main` and `staging` deploys can seed one idempotent bootstrap admin through `npm run db:seed:bootstrap-admin`.
- The scaffolder collects `BOOTSTRAP_ADMIN_EMAIL` and `BOOTSTRAP_ADMIN_PASSWORD` during `new` runs for this template. The email defaults to `admin@example.com`, and the password can be provided or generated by the scaffolder.
- The scaffold stores those bootstrap-admin values in `.scaffold/runtime/secret-context.json` and syncs them into GitHub Actions secrets during GitHub provisioning.
- Demo credentials stay local and staging-only. Production should rely on the scaffold-time bootstrap-admin contract instead of `admin@example.com / password123`.

## Current known Phase 0 gaps

- API and auth contracts are only partially hardened; see `docs/GAP_ANALYSIS.md` before widening runtime behavior.
- mobile dashboard and admin navigation still need a better small-screen pattern.
- backend lint, typecheck, and build scripts are still unavailable.

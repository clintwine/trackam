# Frontend Dev

## Purpose
Implement frontend changes without drifting from the current user and admin dashboard baseline.

## Read first
- AGENTS.md
- frontend/AGENTS.md
- docs/UI_ARCHITECTURE.md
- frontend/docs/DESIGN_SYSTEM.md
- frontend/intelligence.md
- brand/BRAND_CONSTRAINTS.md
- docs/API_SPEC.md when the screen is data-backed

## Focus
- auth UI
- user dashboard UI
- admin dashboard UI
- loading, empty, and error states
- mobile navigation gaps
- copy cleanup

## Rules
- Follow existing shared primitives where possible.
- Treat `frontend/intelligence.md` as the reusable frontend guidance layer.
- Treat `brand/BRAND_CONSTRAINTS.md` as the visual overlay.
- Do not assume storefront or commerce UI exists here.
- Keep output concise.

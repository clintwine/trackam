# Constraint Checklist

## Capture from sources
- brand adjectives and anti-adjectives
- color direction
- typography direction
- imagery style
- iconography and shape language
- motion tone
- accessibility or clarity expectations
- explicit do and do not rules

## Separate carefully
| Belongs in | Put here when |
|---|---|
| `brand/BRAND_CONSTRAINTS.md` | The rule is brand-specific or project-specific |
| `frontend/intelligence.md` | The rule becomes reusable across multiple current screens or shells |
| local UI notes | The rule is only for one reference batch or one screen |

## Mark as inferred when
- the source suggests a direction but does not state it directly
- multiple references imply a consistent pattern
- the repo needs a temporary bridge until better source material exists
